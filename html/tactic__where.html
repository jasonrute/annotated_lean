 10  # The `where` command
 11  
 12  When working in a Lean file with namespaces, parameters, and variables,
 13  it can be confusing to identify what the current "parser context" is.
 14  The command `#where` tries to identify and print information about the current location,
 15  including the active namespace, open namespaces, and declared variables.
 16  
 17  This information is not "officially" accessible in the metaprogramming environment;
 18  `#where` retrieves it via a number of hacks that are not always reliable.
 19  While it is very useful as a quick reference, users should not assume its output is correct.
 20  -/
