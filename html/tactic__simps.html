118  /--
119  * Automatically derive lemmas specifying the projections of this declaration.
120  * Example: (note that the forward and reverse functions are specified differently!)
121    ```
122    @[simps] def refl (α) : α ≃ α := ⟨id, λ x, x, λ x, rfl, λ x, rfl⟩
123    ```
124    derives two simp-lemmas:
125    ```
126    @[simp] lemma refl_to_fun (α) (x : α) : (refl α).to_fun x = id x
127    @[simp] lemma refl_inv_fun (α) (x : α) : (refl α).inv_fun x = x
128    ```
129  * It does not derive simp-lemmas for the prop-valued projections.
130  * It will automatically reduce newly created beta-redexes, but not unfold any definitions.
131  * If one of the fields itself is a structure, this command will recursively create
132    simp-lemmas for all fields in that structure.
133  * You can use `@[simps proj1 proj2 ...]` to only generate the projection lemmas for the specified
134    projections. For example:
135    ```
136    attribute [simps to_fun] refl
137    ```
138  * If one of the values is an eta-expanded structure, we will eta-reduce this structure.
139  * You can use `@[simps lemmas_only]` to derive the lemmas, but not mark them
140    as simp-lemmas.
141  * You can use `@[simps short_name]` to only use the name of the last projection for the name of the
142    generated lemmas.
143  * The precise syntax is `('simps' 'lemmas_only'? 'short_name'? ident*)`.
144  * If one of the projections is marked as a coercion, the generated lemmas do *not* use this
145    coercion.
146  * `@[simps]` reduces let-expressions where necessary.
147  * If one of the fields is a partially applied constructor, we will eta-expand it
148    (this likely never happens).
149    -/
