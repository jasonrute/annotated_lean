 12  # The category of elements
 13  
 14  This file defines the category of elements, also known as (a special case of) the Grothendieck construction.
 15  
 16  Given a functor `F : C ⥤ Type`, an object of `F.elements` is a pair `(X : C, x : F.obj X)`.
 17  A morphism `(X, x) ⟶ (Y, y)` is a morphism `f : X ⟶ Y` in `C`, so `F.map f` takes `x` to `y`.
 18  
 19  ## Implementation notes
 20  This construction is equivalent to a special case of a comma construction, so this is mostly just
 21  a more convenient API. We prove the equivalence in `category_theory.category_of_elements.comma_equivalence`.
 22

 41  /-- The category structure on `F.elements`, for `F : C ⥤ Type`.
 42      A morphism `(X, x) ⟶ (Y, y)` is a morphism `f : X ⟶ Y` in `C`, so `F.map f` takes `x` to `y`.
 43   -/
 44  instance category_of_elements (F : C ⥤ Type u) : category F.elements :=
 45  { hom := λ p q, { f : p.1 ⟶ q.1 // (F.map f) p.2 = q.2 },
 46    id := λ p, ⟨𝟙 p.1, by obviously⟩,
 47    comp := λ p q r f g, ⟨f.val ≫ g.val, by obviously⟩ }

 80  /-- The equivalence between the category of elements `F.elements`
 81      and the comma category `(*, F)`. -/
 82  def comma_equivalence : F.elements ≌ comma (functor.of.obj punit) F :=
 83  equivalence.mk (to_comma F) (from_comma F)
