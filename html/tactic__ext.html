 12  /--
 13  `derive_struct_ext_lemma n` generates two extensionality lemmas based on
 14  the equality of all non-propositional projections.
 15  
 16  On the following:
 17  
 18  ```
 19  @[ext]
 20  structure foo (α : Type*) :=
 21  (x y : ℕ)
 22  (z : {z // z < x})
 23  (k : α)
 24  (h : x < y)
 25  ```
 26  
 27  `derive_struct_lemma` generates:
 28  
 29  ```
 30  lemma foo.ext : ∀ {α : Type u_1} (x y : foo α), x.x = y.x → x.y = y.y → x.z == y.z → x.k = y.k → x = y
 31  lemma foo.ext_iff : ∀ {α : Type u_1} (x y : foo α), x = y ↔ x.x = y.x ∧ x.y = y.y ∧ x.z == y.z ∧ x.k = y.k
 32  ```

151   Tag lemmas of the form:
152  
153   ```
154   @[ext]
155   lemma my_collection.ext (a b : my_collection)
156     (h : ∀ x, a.lookup x = b.lookup y) :
157     a = b := ...
158   ```
159  
160   The attribute indexes extensionality lemma using the type of the
161   objects (i.e. `my_collection`) which it gets from the statement of
162   the lemma.  In some cases, the same lemma can be used to state the
163   extensionality of multiple types that are definitionally equivalent.
164  
165   ```
166   attribute [ext [(→),thunk,stream]] funext
167   ```
168  
169   Those parameters are cumulative. The following are equivalent:
170  
171   ```
172   attribute [ext [(→),thunk]] funext
173   attribute [ext [stream]] funext
174   ```
175   and
176   ```
177   attribute [ext [(→),thunk,stream]] funext
178   ```
179  
180   One removes type names from the list for one lemma with:
181   ```
182   attribute [ext [-stream,-thunk]] funext
183    ```
184  
185   Also, the following:
186  
187   ```
188   @[ext]
189   lemma my_collection.ext (a b : my_collection)
190     (h : ∀ x, a.lookup x = b.lookup y) :
191     a = b := ...
192   ```
193  
194   is equivalent to
195  
196   ```
197   @[ext *]
198   lemma my_collection.ext (a b : my_collection)
199     (h : ∀ x, a.lookup x = b.lookup y) :
200     a = b := ...
201   ```
202  
203   This allows us specify type synonyms along with the type
204   that referred to in the lemma statement.
205  
206   ```
207   @[ext [*,my_type_synonym]]
208   lemma my_collection.ext (a b : my_collection)
209     (h : ∀ x, a.lookup x = b.lookup y) :
210     a = b := ...
211   ```
