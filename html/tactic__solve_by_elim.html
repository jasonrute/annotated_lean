 48  /--
 49  Configuration options for `solve_by_elim`.
 50  
 51  * By default `solve_by_elim` operates only on the first goal,
 52    but with `backtrack_all_goals := true`, it operates on all goals at once,
 53    backtracking across goals as needed,
 54    and only succeeds if it dischargers all goals.
 55  * `discharger` specifies a tactic to try discharge subgoals
 56    (this is only attempted on subgoals for which no lemma applies successfully).
 57  * `assumptions` generates the list of lemmas to use in the backtracking search.
 58  * `max_rep` bounds the depth of the search.

 83  /--
 84  `apply_assumption` looks for an assumption of the form `... → ∀ _, ... → head`
 85  where `head` matches the current goal.
 86  
 87  alternatively, when encountering an assumption of the form `sg₀ → ¬ sg₁`,
 88  after the main approach failed, the goal is dismissed and `sg₀` and `sg₁`
 89  are made into the new goal.
 90  
 91  optional arguments:
 92  - asms: list of rules to consider instead of the local constants
 93  - tac:  a tactic to run on each subgoals after applying an assumption; if
 94          this tactic fails, the corresponding assumption will be rejected and
 95          the next one will be attempted.
 96  -/

102  /--
103  `solve_by_elim` calls `apply` on the main goal to find an assumption whose head matches
104  and then repeatedly calls `apply` on the generated subgoals until no subgoals remain,
105  performing at most `max_rep` recursive steps.
106  
107  `solve_by_elim` discharges the current goal or fails.
108  
109  `solve_by_elim` performs back-tracking if `apply_assumption` chooses an unproductive assumption.
110  
111  By default, the assumptions passed to apply_assumption are the local context, `rfl`, `trivial`, `congr_fun` and
112  `congr_arg`.
113  
114  `solve_by_elim [h₁, h₂, ..., hᵣ]` also applies the named lemmas.
115  
116  `solve_by_elim with attr₁ ... attrᵣ also applied all lemmas tagged with the specified attributes.
117  
118  `solve_by_elim only [h₁, h₂, ..., hᵣ]` does not include the local context, `rfl`, `trivial`, `congr_fun`, or `congr_arg`
119  unless they are explicitly included.
120  
121  `solve_by_elim [-id]` removes a specified assumption.
122  
123  `solve_by_elim*` tries to solve all goals together, using backtracking if a solution for one goal
124  makes other goals impossible.
125  
126  optional arguments:
127  - discharger: a subsidiary tactic to try at each step (e.g. `cc` may be helpful)
128  - max_rep: number of attempts at discharging generated sub-goals
129  -/
