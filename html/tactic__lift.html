133  /-- Lift an expression to another type.
134    * Usage: `'lift' expr 'to' expr ('using' expr)? ('with' id (id id?)?)?`.
135    * If `n : ℤ` and `hn : n ≥ 0` then the tactic `lift n to ℕ using hn` creates a new
136      constant of type ℕ, also named `n` and replaces all occurrences of the old variable `(n : ℤ)`
137      with `↑n` (where `n` in the new variable). It will remove `n` and `hn` from the context.
138    * The argument `using hn` is optional, the tactic `lift n to ℕ` does the same, but also creates a
139      new subgoal that `n ≥ 0` (where `n` is the old variable).
140    * You can also use `lift n to ℕ using e` where `e` is any expression of type `n ≥ 0`.
141    * Use `lift n to ℕ with k` to specify the name of the new variable.
142    * Use `lift n to ℕ with k hk` to also specify the name of the equality `↑k = n`. In this case, `n`
143      will remain in the context. You can use `rfl` for the name of `hk` to substitute it away.
144    * You can also use `lift e to ℕ with k hk` where `e` is any expression of type `ℤ`.
145      In this case, the `hk` will always stay in the context, but it will be used to rewrite `e` in
146      all hypotheses and the target.
147    * The tactic `lift n to ℕ using h` will remove `h` from the context. If you want to keep it,
148      specify it again as the third argument to `with`, like this: `lift n to ℕ using h with n rfl h`.
149    * More generally, this can lift an expression from `α` to `β` assuming that there is an instance
150      of `can_lift α β`. In this case the proof obligation is specified by `can_lift.cond`.
151    * Given an instance `can_lift β γ`, it can also lift `α → β` to `α → γ`, and similarly for
152      general pi types, see `pi.can_lift`.
