 26  /-- `compact_relation bs as_ps`: Produce a relation of the form:
 27    R as := ∃ bs, Λ_i a_i = p_i[bs]
 28  This relation is user visible, so we compact it by removing each `b_j` where a `p_i = b_j`, and
 29  hence `a_i = b_j`. We need to take care when there are `p_i` and `p_j` with `p_i = p_j = b_k`.

138  /-- `mk_iff_of_inductive_prop i r` makes a iff rule for the inductively defined proposition `i`.
139    The new rule `r` has the shape `∀ps is, i as ↔ ⋁_j, ∃cs, is = cs`, where `ps` are the type
140    parameters, `is` are the indices, `j` ranges over all possible constructors, the `cs` are the
141    parameters for each constructors, the equalities `is = cs` are the instantiations for each
142    constructor for each of the indices to the inductive type `i`.
143  
144    In each case, we remove constructor parameters (i.e. `cs`) when the corresponding equality would
145    be just `c = i` for some index `i`.
146  
147    For example: `mk_iff_of_inductive_prop` on `list.chain` produces:
148  
149      ∀{α : Type*} (R : α → α → Prop) (a : α) (l : list α),
150        chain R a l ↔ l = [] ∨ ∃{b : α} {l' : list α}, R a b ∧ chain R b l ∧ l = b :: l'
151  
152  -/
