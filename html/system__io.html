208  /--
209  This is the "back door" into the `io` monad, allowing IO computation to be performed during tactic execution.
210  For this to be safe, the IO computation should be ideally free of side effects and independent of its environment.
211  This primitive is used to invoke external tools (e.g., SAT and SMT solvers) from a tactic.
212  
213  IMPORTANT: this primitive can be used to implement `unsafe_perform_io {α : Type} : io α → option α`
214  or `unsafe_perform_io {α : Type} [inhabited α] : io α → α`. This can be accomplished by executing
215  the resulting tactic using an empty `tactic_state` (we have `tactic_state.mk_empty`).
216  If `unsafe_perform_io` is defined, and used to perform side-effects, users need to take the following
217  precautions:
218  
219  - Use `@[noinline]` attribute in any function to invokes `tactic.unsafe_perform_io`.
220    Reason: if the call is inlined, the IO may be performed more than once.
221  
222  - Set `set_option compiler.cse false` before any function that invokes `tactic.unsafe_perform_io`.
223    This option disables common subexpression elimination. Common subexpression elimination
224    might combine two side effects that were meant to be separate.
225
