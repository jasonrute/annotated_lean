 13  /-- The type of objects of the opposite of `α`; used to defined opposite category/group/...
 14  
 15    In order to avoid confusion between `α` and its opposite type, we
 16    set up the type of objects `opposite α` using the following pattern,
 17    which will be repeated later for the morphisms.
 18  
 19    1. Define `opposite α := α`.
 20    2. Define the isomorphisms `op : α → opposite α`, `unop : opposite α → α`.
 21    3. Make the definition `opposite` irreducible.
 22  
 23    This has the following consequences.
 24  
 25    * `opposite α` and `α` are distinct types in the elaborator, so you
 26      must use `op` and `unop` explicitly to convert between them.
 27    * Both `unop (op X) = X` and `op (unop X) = X` are definitional
 28      equalities. Notably, every object of the opposite category is
 29      definitionally of the form `op X`, which greatly simplifies the
 30      definition of the structure of the opposite category, for example.
 31  
 32    (If Lean supported definitional eta equality for records, we could
 33    achieve the same goals using a structure with one field.)
 34  -/
