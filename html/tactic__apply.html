  8  /-!
  9  This file provides an alternative implementation for `apply` to fix the so-called "apply bug".
 10  
 11  The issue arises when the goals is a Π-type -- whether it is visible or hidden behind a definition.
 12  
 13  For instance, consider the following proof:
 14  
 15  ```
 16  example {α β} (x y z : α → β) (h₀ : x ≤ y) (h₁ : y ≤ z) : x ≤ z :=
 17  begin
 18    apply le_trans,
 19  end
 20  ```
 21  
 22  Because `x ≤ z` is definitionally equal to `∀ i, x i ≤ z i`, `apply` will fail. The alternative definition,
 23  `apply'` fixes this. When `apply` would work, `apply` is used and otherwise, a different strategy is deployed
 24  -/

118  /--
119  Similarly to `apply`, the `apply'` tactic tries to match the current goal against the conclusion of the type of term.
120  
121  It differs from `apply` in that it does not unfold definition in order to find out what the assumptions of the provided term is. It is especially useful when defining relations on function spaces (e.g. `≤`) so that rules like transitivity on `le : (α → β) → (α → β) → (α → β)` will be considered to have three parameters and two assumptions (i.e. `f g h : α → β`, `H₀ : f ≤ g`, `H₁ : g ≤ h`) instead of three parameters, two assumptions and then one more parameter (i.e. `f g h : α → β`, `H₀ : f ≤ g`, `H₁ : g ≤ h`, `x : α`). Whereas `apply` would expect the goal `f x ≤ h x`, `apply'` will work with the goal `f ≤ h`.
122  -/
