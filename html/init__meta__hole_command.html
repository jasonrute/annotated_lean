  9  /--
 10  The front-end (e.g., Emacs, VS Code) can invoke commands for holes {! ... !} in
 11  a declaration. A command is a tactic that takes zero or more pre-terms in the
 12  hole, and returns a list of pair (s, descr) where 's' is a substitution and 'descr' is
 13  a short explanation for the substitution.
 14  Each string 's' represents a different way to fill the hole.
 15  The front-end is responsible for replacing the hole with the string/alternative selected by the user.
 16  
 17  This infra-structure can be use to implement auto-fill and/or refine commands.
 18  
 19  An action may return an empty list. This is useful for actions that just return
 20  information such as: the type of an expression, its normal form, etc.
 21  -/
