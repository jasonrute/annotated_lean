 31  library_note "locally reducible category instances"
 32  "We make SemiRing (and the other categories) locally reducible in order
 33  to define its instances. This is because writing, for example,
 34  
 35  ```
 36  instance : concrete_category SemiRing := by { delta SemiRing, apply_instance }
 37  ```
 38  
 39  results in an instance of the form `id (bundled_hom.concrete_category _)`
 40  and this `id`, not being [reducible], prevents a later instance search
 41  (once SemiRing is no longer reducible) from seeing that the morphisms of
 42  SemiRing are really semiring morphisms (`→+*`), and therefore have a coercion
 43  to functions, for example. It's especially important that the `has_coe_to_sort`
 44  instance not contain an extra `id` as we want the `semiring ↥R` instance to
 45  also apply to `semiring R.α` (it seems to be impractical to guarantee that
 46  we always access `R.α` through the coercion rather than directly)."
