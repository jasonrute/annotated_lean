  7  /-
  8  The elaborator tries to insert coercions automatically.
  9  Only instances of has_coe type class are considered in the process.
 10  
 11  Lean also provides a "lifting" operator: ↑a.
 12  It uses all instances of has_lift type class.
 13  Every has_coe instance is also a has_lift instance.
 14  
 15  We recommend users only use has_coe for coercions that do not produce a lot
 16  of ambiguity.
 17  
 18  All coercions and lifts can be identified with the constant coe.
 19  
 20  We use the has_coe_to_fun type class for encoding coercions from
 21  a type to a function space.
 22  
 23  We use the has_coe_to_sort type class for encoding coercions from
 24  a type to a sort.
 25  -/

100  /- We add this instance directly into has_coe_t to avoid non-termination.
101  
102     Suppose coe_option had type (has_coe a (option a)).
103     Then, we can loop when searching a coercion from α to β (has_coe_t α β)
104     1- coe_trans at (has_coe_t α β)
