 14  # Single-object category
 15  
 16  Single object category with a given monoid of endomorphisms.  It is defined to facilitate transfering
 17  some definitions and lemmas (e.g., conjugacy etc.) from category theory to monoids and groups.
 18  
 19  ## Main definitions
 20  
 21  Given a type `α` with a monoid structure, `single_obj α` is `unit` type with `category` structure
 22  such that `End (single_obj α).star` is the monoid `α`.  This can be extended to a functor `Mon ⥤
 23  Cat`.
 24  
 25  If `α` is a group, then `single_obj α` is a groupoid.
 26  
 27  An element `x : α` can be reinterpreted as an element of `End (single_obj.star α)` using
 28  `single_obj.to_End`.
 29  
 30  ## Implementation notes
 31  
 32  - `category_struct.comp` on `End (single_obj.star α)` is `flip (*)`, not `(*)`. This way
 33    multiplication on `End` agrees with the multiplication on `α`.
 34  
 35  - By default, Lean puts instances into `category_theory` namespace instead of
 36    `category_theory.single_obj`, so we give all names explicitly.
 37  -/

 77  /-- There is a 1-1 correspondence between monoid homomorphisms `α → β` and functors between the
 78      corresponding single-object categories. It means that `single_obj` is a fully faithful
 79      functor. -/
 80  def map_hom (α : Type u) (β : Type v) [monoid α] [monoid β] :
 81    (α →* β) ≃ (single_obj α) ⥤ (single_obj β) :=
 82  { to_fun := λ f,
 83    { obj := id,
 84      map := λ _ _, ⇑f,
 85      map_id' := λ _, f.map_one,
 86      map_comp' := λ _ _ _ x y, f.map_mul y x },
 87    inv_fun := λ f,
 88      { to_fun := @functor.map _ _ _ _ f (single_obj.star α) (single_obj.star α),
 89        map_one' := f.map_id _,
 90        map_mul' := λ x y, f.map_comp y x },
 91    left_inv := λ ⟨f, h₁, h₂⟩, rfl,
 92    right_inv := λ f, by cases f; obviously }
