 53  /--
 54    specialized is true if the result of fun_info has been specifialized
 55    using this argument.
 56    For example, consider the function
 57  
 58               f : Pi (α : Type), α -> α
 59  
 60    Now, suppse we request get_specialize fun_info for the application
 61  
 62           f unit a
 63  
 64    fun_info_manager returns two param_info objects:
 65    1) specialized = true
 66    2) is_subsingleton = true
 67  
 68    Note that, in general, the second argument of f is not a subsingleton,
 69    but it is in this particular case/specialization.
 70  
 71    \remark This bit is only set if it is a dependent parameter.
 72  
 73     Moreover, we only set is_specialized IF another parameter
 74     becomes a subsingleton -/
