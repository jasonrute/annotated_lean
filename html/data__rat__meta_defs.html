 10  # Meta operations on ℚ
 11  
 12  This file defines functions for dealing with rational numbers as expressions.
 13  
 14  They are not defined earlier in the hierarchy, in the `tactic` or `meta` folders, since we do not
 15  want to import `data.rat.basic` there.
 16  
 17  ## Main definitions
 18  
 19  * `rat.mk_numeral` embeds a rational `q` as a numeral expression into a type supporting the needed
 20    operations. It does not need a tactic state.
 21  * `rat.reflect` specializes `rat.mk_numeral` to `ℚ`.
 22  * `expr.of_rat` behaves like `rat.mk_numeral`, but uses the tactic state to infer the needed
 23    structure on the target type.
 24  
 25  * `expr.to_rat` evaluates a normal numeral expression as a rat.
 26  * `expr.eval_rat` evaluates a numeral expression with arithmetic operations as a rat.
 27  
 28  -/
 29  
 30  /--
 31  `rat.mk_numeral q` embeds `q` as a numeral expression inside a type with 0, 1, +, -, and /
 32  
 33  `type`: an expression representing the target type. This must live in Type 0.
 34  `has_zero`, `has_one`, `has_add`: expressions of the type `has_zero %%type`, etc.
 35  
 36  This function is similar to `expr.of_rat` but takes more hypotheses and is not tactic valued.
 37   -/
