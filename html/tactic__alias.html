 10  # The `alias` command
 11  
 12  This file defines an `alias` command, which can be used to create copies
 13  of a theorem or definition with different names.
 14  
 15  Syntax:
 16  
 17  ```
 18  / -- doc string - /
 19  alias my_theorem ← alias1 alias2 ...
 20  ```
 21  
 22  This produces defs or theorems of the form:
 23  
 24  ```
 25  / -- doc string - /
 26  @[alias] theorem alias1 : <type of my_theorem> := my_theorem
 27  
 28  / -- doc string - /
 29  @[alias] theorem alias2 : <type of my_theorem> := my_theorem
 30  ```
 31  
 32  Iff alias syntax:
 33  
 34  ```
 35  alias A_iff_B ↔ B_of_A A_of_B
 36  alias A_iff_B ↔ ..
 37  ```
 38  
 39  This gets an existing biconditional theorem `A_iff_B` and produces
 40  the one-way implications `B_of_A` and `A_of_B` (with no change in
 41  implicit arguments). A blank `_` can be used to avoid generating one direction.
 42  The `..` notation attempts to generate the 'of'-names automatically when the
 43  input theorem has the form `A_iff_B` or `A_iff_B_left` etc.
 44  -/
