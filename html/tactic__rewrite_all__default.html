  6  -- There are two alternative backends, provided by `.congr` and `.kabstract`.
  7  --
  8  -- The kabstract backend is faster, but if there are multiple identical occurences of the
  9  -- same rewritable subexpression, all are rewritten simultaneously, and this isn't always
 10  -- what we want.
 11  -- (In particular, rewrite_search is much less capable on the category_theory library.)

 62  /--
 63  `perform_nth_write n rules` performs only the `n`th possible rewrite using the `rules`.

 65  The core `rewrite` has a `occs` configuration setting intended to achieve a similar
 66  purpose, but this doesn't really work. (If a rule matches twice, but with different
 67  values of arguments, the second match will not be identified.)
